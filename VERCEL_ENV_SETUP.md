# Настройка переменных окружения в Vercel

Для корректной работы Telegram бота в продакшн среде необходимо настроить следующие переменные окружения в Vercel:

## Необходимые переменные окружения

1. **TELEGRAM_BOT_TOKEN** - токен вашего Telegram бота
   - Значение: получите в BotFather

2. **TELEGRAM_CHAT_ID** - ID чата для отправки уведомлений
   - Значение: `592052544`

## Как настроить в Vercel Dashboard

1. Перейдите в проект на [Vercel Dashboard](https://vercel.com)
2. Выберите ваш проект `intermag`
3. Перейдите в раздел **Settings** → **Environment Variables**
4. Добавьте следующие переменные:

   ```
   TELEGRAM_BOT_TOKEN=your_telegram_bot_token
   TELEGRAM_CHAT_ID=your_telegram_chat_id
   ```

5. Убедитесь, что переменные добавлены для **Production Environment**
6. Нажмите **Save**
7. Перезапустите деплоймент (Redeploy)

## Альтернативный способ: через Vercel CLI

Если у вас установлен Vercel CLI, вы можете добавить переменные через командную строку:

```bash
vercel env add TELEGRAM_BOT_TOKEN
# Введите значение: ваш токен бота

vercel env add TELEGRAM_CHAT_ID
# Введите значение: ID вашего чата

# Для продакшн среды
vercel env pull .env.production
```

## Проверка работы

После настройки переменных окружения:

1. Сделайте новый деплоймент
2. Протестируйте оформление заказа на продакшн сайте
3. Проверьте, что уведомления приходят в Telegram

## Устранение неполадок

Если уведомления не приходят:

1. Проверьте, что переменные окружения установлены правильно
2. Убедитесь, что бот имеет доступ к указанному чату
3. Проверьте логи Vercel в разделе **Functions**
4. Убедитесь, что бот активен и не заблокирован

## Безопасность

Токен бота является конфиденциальной информацией. Не коммитьте его в публичные репозитории. Файлы `.env.local` и `.env.production` добавлены в `.gitignore` для безопасности.
